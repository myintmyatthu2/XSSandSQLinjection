CSRF (Cross-Site Request Forgery) Tutorial

Warning: This application contains intentional CSRF vulnerabilities for educational purposes.

What is CSRF?

Cross-Site Request Forgery (CSRF) is an attack that forces an end user to execute unwanted actions on a web application in which they're currently authenticated.

How CSRF Works

Attackers trick users into submitting requests to a website where they are authenticated, causing unintended actions.

Try It Out

First, disable CSRF protection in the Security Dashboard.

Then create an HTML file with this content:

<form action="http://localhost:5000/buy" method="POST">
  <input type="hidden" name="product_id" value="1">
  <input type="hidden" name="price" value="1">
</form>
<script>document.forms[0].submit();</script>

Open this file while logged into the application to see the CSRF attack in action.

Prevention

  • Use CSRF tokens
  • Check Origin and Referer headers
  • Use SameSite cookie attribute
  • Require re-authentication for sensitive actions

Toggle CSRF protection to see how it prevents these attacks.
